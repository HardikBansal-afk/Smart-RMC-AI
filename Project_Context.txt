# Intelligent Document Processing (IDP) Pipeline

## Project Overview

This project extracts unstructured engineering data such as Bill of Materials (BOM) and Parts Lists from PDF documents and converts them into a structured Excel database using a Hybrid Slicer architecture.

---

## Pipeline Architecture

### Phase 1 – Ingestion & Filtering

* Uses **pdfplumber** to extract engineering tables while preserving their spatial layout.
* A **Finite State Automata (DFA)** filters unwanted headers and irrelevant rows.
* A **Regex Demultiplexer** separates grouped components (e.g., "Pump A/B") into individual records.

### Phase 2 – Memory & Structuring

* A **Quadratic Probing Hash Table** removes duplicate components.
* An **N-ary Tree** preserves equipment hierarchy by attaching extracted parts to their corresponding parent equipment.

### Phase 3 – Cloud Map-Reduce

* Structured records are divided into micro-batches of **15 items**.
* Each batch is processed using the **Groq API** with the **llama-3.1-8b-instant** model.
* **Instructor** and **Pydantic** enforce a strict JSON response schema.
* Model temperature is set to **0.0** for deterministic outputs.

### Phase 4 – Fault Tolerance

* Every successful batch is automatically saved to **temp_checkpoint.csv**.
* If a **429 Rate Limit Error** occurs, the pipeline waits for **60 seconds** and resumes from the latest checkpoint.

## Required Libraries

Install the required dependencies using:

pip install pdfplumber pandas openai pydantic instructor openpyxl

---

## Configuration

Before running the pipeline, update the following variables inside the `if __name__ == "__main__":` block:

1. Replace `YOUR_API_KEY` with your active Groq API key.
2. Set `folder_path` to the directory containing the input PDF files.
3. Set `output_path` to the desired location of the output Excel file.

> **Windows users:** Use double backslashes (`\\`) in file paths.

---

## Running the Pipeline

Open a terminal, navigate to the project directory, and run:

python idp_cloud_pipeline.py

---

## Output

The pipeline will:

* Read all PDF files from the input folder.
* Extract engineering tables.
* Filter unwanted content.
* Split grouped components.
* Remove duplicate records.
* Preserve equipment hierarchy.
* Process data using the Groq API.
* Save progress after each batch.
* Resume automatically after rate limits.
* Generate the final Excel BOM database.
